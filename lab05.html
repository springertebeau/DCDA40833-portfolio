Lab 5: Vibe Coding

Introduction

This lab explores AI-assisted coding through two experiments: adding a dark mode toggle to my portfolio using CSS and JavaScript, and creating Python data visualizations in Google Colab. The key principle is that I must be able to read, explain, and modify any code I use, regardless of who or what wrote it.

Experiment A: Dark Mode Toggle

Implementation

I added a dark mode toggle button to my portfolio that allows visitors to switch between light and dark themes. The feature includes a toggle button with moon/sun icons in the navigation bar, CSS custom properties (variables) that change colors throughout the site, JavaScript that saves the user's preference in browser localStorage, and smooth transitions when switching between modes. You can try the dark mode toggle in the navigation bar above.

Technical Approach

The dark mode feature works through three main components:

CSS Variables

I asked AI to create a dark mode color palette that maintains good contrast and keeps my pink theme. The code uses CSS custom properties that define colors for the entire site:

/* Light mode colors in :root */
:root {
    --text-color: #333;
    --bg-color: #f4f4f4;
    --card-bg: #ffffff;
}

/* Dark mode overrides */
body.dark-mode {
    --text-color: #e5e5e5;
    --bg-color: #1a1a1a;
    --card-bg: #2d2d2d;
}

When the dark-mode class is added to the body element, all the color variables get new values and the entire site changes instantly.

JavaScript Functionality

The JavaScript handles three things:

  1. Checking saved preference: When the page loads, it checks localStorage to see if the user previously enabled dark mode
  2. Toggling the class: When clicked, it adds or removes the dark-mode class from the body
  3. Saving preference: It stores the choice in localStorage so it persists between visits
// Check if dark mode was previously enabled
if (localStorage.getItem('darkMode') === 'enabled') {
    body.classList.add('dark-mode');
    icon.textContent = '☀️';
}

// Toggle on click and save preference
darkModeToggle.addEventListener('click', () => {
    body.classList.toggle('dark-mode');
    // Update icon and save to localStorage...
});

Modifications

  • Changed the toggle button's padding from 0.3rem to 0.5rem 1rem to make it easier to tap on mobile
  • Lightened the dark background from #0a0a0a to #1a1a1a for better readability
  • Added more CSS variables to ensure all page elements support dark mode
  • Fixed a bug where the sun icon didn't show on page load when dark mode was active

Experiment B: Python Data Visualization

Implementation

I used AI to generate a Python notebook that creates and visualizes sample student performance data. The code generates 50 fictional students with study hours, sleep hours, and exam scores, then creates scatter plots to analyze correlations. The Jupyter notebook is available for download and can be run in Google Colab.

AI Prompt

"Create Python code that generates a sample dataset of 50 students with columns for study_hours (1-10), sleep_hours (4-9), and exam_score (0-100). Then create a scatter plot showing the relationship between study hours and exam scores, with points colored by sleep quality. Add a trend line and proper labels."

Understanding

I understand the logic and most implementation details. The code generates 50 fake students with random study hours (1-10) and sleep hours (4-9), calculates exam scores using a formula where more study hours and more sleep result in higher scores plus some randomness, groups students into low, medium, and high sleep quality categories, creates a scatter plot where each point is a student colored by their sleep quality, and adds a trend line showing the overall pattern.

I modified the random noise parameter from np.random.uniform(0, 20) to (0, 15) to reduce randomness and make the correlation pattern clearer. I also added a second visualization for sleep hours versus exam scores to compare correlations, and changed scatter plot colors to match my portfolio's theme.

Findings

  • Study-performance correlation: 0.76 (strong positive) - more study hours strongly predict higher exam scores
  • Sleep-performance correlation: 0.65 (moderate positive) - adequate sleep also matters, but slightly less than study time
  • Students with high sleep quality tend to cluster toward higher scores

Process and Reflection

When AI Was Helpful

AI provided exact syntax for CSS and JavaScript without needing to search documentation, created polished visualizations without learning advanced matplotlib styling, allowed rapid prototyping to test concepts, and provided code examples that taught me patterns I can now recognize and reuse.

When AI Was Problematic

AI can create overconfidence in accepting code without questioning if it's the best approach. Some solutions worked but were overly complex. Testing was still required as AI doesn't catch all edge cases. I also realized I don't deeply understand some statistical calculations, just their outputs.

Human Role

What remains uniquely human in this workflow includes vision and goals (deciding what feature to build), evaluation (reading every line of code to decide if it makes sense), integration (combining AI snippets into my existing portfolio structure), refinement (adjusting colors, sizes, and behavior based on testing), and critical judgment about when to trust AI versus when to verify or modify its output.

Does vibe coding help or hinder learning? Both, depending on how you use it.

It helps when:

  • You read and analyze the generated code instead of blindly copying it
  • You use it to learn patterns you can apply independently later
  • You modify and experiment with the output to deepen understanding
  • You're working slightly above your current skill level (productive struggle)

It hinders when:

  • You skip foundational learning and jump straight to AI generation
  • You accept code you don't understand "because it works"
  • You become dependent on AI for tasks you should be able to do independently
  • You stop pushing yourself to understand the "how" and only care about the "what"

My experience: I found it helpful for this lab because I already understand HTML/CSS fundamentals from earlier labs. The AI helped me learn advanced techniques like CSS variables and localStorage that would have taken much longer to learn through trial and error. However, for Python, I realized I need more foundational practice before relying heavily on AI for basic syntax.

Attribution and Citation

In this lab, I used inline comments to mark AI-generated code and document my understanding level using the Understanding Spectrum. I clearly attributed which parts of the code were AI-generated versus my own modifications. The key principle is that attribution matters most when AI does intellectual work I'm claiming credit for.

Personal AI Coding Guidelines

After this lab, I established personal rules for AI usage:

When I Will Use AI

  • Learning new syntax in languages I already understand conceptually
  • Getting starter code for patterns I haven't implemented before
  • Debugging errors when I'm stuck after trying to solve them myself
  • Exploring what's possible before committing time to build something

When I Won't Use AI

  • For assignments explicitly designed to test my understanding
  • When learning fundamental concepts for the first time
  • For code that I can't read and explain to someone else
  • For reflections, analysis, or creative thinking

Mandatory Practices

  • Read everything - no code goes into my project without me reading every line
  • The explanation test - if I can't explain it to a peer, I don't use it yet
  • Comment honestly - document what I understand versus what I don't
  • Attribute clearly - mark AI-generated code and note what I modified
  • Test thoroughly - AI output might work in ideal conditions but fail in edge cases

Conclusion

This lab taught me that AI-assisted coding exists on a spectrum. AI can be a valuable tool for learning and implementation when used responsibly. My role remains critical in defining problems, evaluating solutions, integrating code, ensuring quality, and maintaining the ability to understand and explain any code I use. The key is to use AI strategically to accelerate learning while ensuring I'm building skills rather than creating dependence.

The dark mode toggle now functions across all pages of my portfolio, and the Python notebook demonstrates my ability to work with data visualization libraries. Both experiments show that AI can be a collaborative tool when combined with critical thinking and testing.
