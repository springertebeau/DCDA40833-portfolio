Lab 5: Vibe Coding

AI-Assisted Development: Exploring the Spectrum of AI Collaboration

Overview

This lab explores "vibe coding" - using AI as a coding collaborator to generate, debug, and enhance code. The key principle: I must be able to read, explain, and modify any code I use, regardless of who (or what) wrote it.

I completed two experiments to explore different points on the AI coding spectrum:

  • Experiment A: Portfolio Enhancement - Dark Mode Toggle
  • Experiment B: Python Data Visualization in Google Colab

Experiment A: Dark Mode Toggle

What I Built

I added a dark mode toggle button to my portfolio that allows visitors to switch between light and dark themes. The feature includes:

  • A toggle button with moon/sun icons in the navigation bar
  • CSS custom properties (variables) that change colors throughout the site
  • JavaScript that saves the user's preference in browser localStorage
  • Smooth transitions when switching between modes

How It Works

Understanding Level: I understand the logic and can explain it in my own words.

The dark mode feature works through three main components:

1. CSS Variables (styles.css)

I asked AI to create a dark mode color palette that maintains good contrast and keeps my pink theme. The code uses CSS custom properties (variables prefixed with --) that define colors for the entire site:

/* Light mode colors in :root */
:root {
    --text-color: #333;
    --bg-color: #f4f4f4;
    --card-bg: #ffffff;
}

/* Dark mode overrides */
body.dark-mode {
    --text-color: #e5e5e5;
    --bg-color: #1a1a1a;
    --card-bg: #2d2d2d;
}

How this works: When the dark-mode class is added to the body element, all the color variables get new values. Since the rest of the CSS uses these variables (like color: var(--text-color);), the entire site changes instantly. I understand this concept fully - it's like find-and-replace but for colors.

2. Toggle Button (HTML)

The toggle button uses Unicode emoji icons (for dark mode off, for dark mode on). I modified the AI's original positioning to make it work better on mobile devices.

3. JavaScript Functionality

The JavaScript handles three things:

  1. Checking saved preference: When the page loads, it checks localStorage to see if the user previously enabled dark mode
  2. Toggling the class: When clicked, it adds or removes the dark-mode class from the body
  3. Saving preference: It stores the choice in localStorage so it persists between visits
// Check if dark mode was previously enabled
if (localStorage.getItem('darkMode') === 'enabled') {
    body.classList.add('dark-mode');
    icon.textContent = '‚òÄÔ∏è';
}

// Toggle on click and save preference
darkModeToggle.addEventListener('click', () => {
    body.classList.toggle('dark-mode');

});

My understanding: I understand how localStorage works - it's browser storage that persists data between sessions. I understand event listeners and how classList.toggle() adds/removes classes. The part I couldn't have written from scratch is the proper syntax for localStorage, but I could modify it or explain it to someone else.

What I Modified

  • Changed the toggle button's padding from 0.3rem to 0.5rem 1rem to make it easier to tap on mobile
  • Adjusted some dark mode colors - AI's original dark background was too dark (#0a0a0a), so I lightened it to #1a1a1a
  • Added more CSS variables beyond what AI initially created to ensure all page elements support dark mode (tables, borders, etc.)

Testing & Refinement

I tested by:

  • Clicking the toggle multiple times to verify smooth transitions
  • Refreshing the page to confirm localStorage persistence
  • Checking all pages in my portfolio to ensure consistent behavior
  • Testing on mobile to verify touch-friendly button size

Bug I found: Initially, the sun icon didn't show on page load even when dark mode was active. I fixed this by ensuring the icon update happens in the localStorage check, not just in the click handler.

What the Code Does (In My Own Words):

  1. Creates 50 fake students with random study hours (1-10) and sleep hours (4-9)
  2. Calculates exam scores using a formula: more study hours and more sleep = higher scores, plus some randomness to make it realistic
  3. Groups students into "Low", "Medium", and "High" sleep quality categories
  4. Creates a scatter plot where each point is a student, colored by their sleep quality

Process & Critical Reflection

When AI Was Helpful

  • Syntax assistance: AI knew the exact CSS and JavaScript syntax I needed, saving hours of documentation searching
  • Professional output: The visualizations looked polished without needing to learn advanced matplotlib styling
  • Rapid prototyping: I could test the dark mode concept quickly and decide if I wanted to keep it
  • Learning through examples: Reading AI-generated code taught me patterns I can now recognize and reuse

When AI Was Problematic

  • Over-confidence: It's easy to accept AI code without questioning if it's the best approach
  • Hidden complexity: Some solutions worked but were overly complex for what I needed
  • Testing required: AI doesn't catch all edge cases - I still had to test thoroughly
  • Knowledge gaps: I realized I don't deeply understand statistical calculations, just their outputs

My Role in AI-Assisted Workflow

What remains uniquely human:

  • Vision & Goals: AI can't decide what feature to build - I chose dark mode because I wanted to improve user experience
  • Evaluation: I had to read every line of code and decide if it made sense
  • Integration: AI generates isolated snippets - I integrated them into my existing portfolio structure
  • Refinement: AI gave me a starting point, but I adjusted colors, sizes, and behavior based on testing
  • Critical Judgment: Knowing when to trust AI vs when to verify or modify its output

Learning Trade-offs

Does vibe coding help or hinder learning? Both, depending on how you use it.

It helps when:

  • You read and analyze the generated code instead of blindly copying it
  • You use it to learn patterns you can apply independently later
  • You modify and experiment with the output to deepen understanding
  • You're working slightly above your current skill level (productive struggle)

It hinders when:

  • You skip foundational learning and jump straight to AI generation
  • You accept code you don't understand "because it works"
  • You become dependent on AI for tasks you should be able to do independently
  • You stop pushing yourself to understand the "how" and only care about the "what"

My experience: I found it helpful for this lab because I already understand HTML/CSS fundamentals from earlier labs. The AI helped me learn advanced techniques (CSS variables, localStorage) that would have taken much longer to learn through trial and error. However, for Python, I realized I need more foundational practice before vibe coding - I'm relying too heavily on AI for basic syntax.

Key principle: Attribution matters most when the AI does intellectual work I'm claiming credit for. If I use AI the way I'd use a reference book - to look up syntax or patterns I then integrate myself - attribution is less critical than ensuring correctness.
