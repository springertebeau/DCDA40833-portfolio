Code Explanation

The Python script follows this workflow:

  1. Read the CSV file using pandas
  2. For each location, send the address to Mapbox Geocoding API to get lat/long coordinates
  3. Create a Folium map centered on my hometown with my custom Mapbox basemap
  4. Add markers with different colors based on location type
  5. Create HTML pop-ups with location name, description, and image
  6. Save the map as an HTML file

The complete Python script is available in hometown_map.py.

Written Reflection

Lab 6 focused on designing a basemap and creating an interactive map that works across different zoom levels. The main inspiration for my basemap design came from looking at modern digital maps that use simple colors and clean visual styles. Many modern maps focus on clarity and readability, which means avoiding too many bright colors or unnecessary visual elements. I wanted my map to have a similar clean look so that the important geographic features would stand out.

To translate this inspiration into my own map, I focused on using a limited color palette and consistent styling. For example, land areas were kept neutral while water features used cooler colors so they were easy to identify. Roads and other features were styled in ways that made them visible but not overwhelming. The goal was to create a map that looked visually balanced and easy to read.

One of the main cartographic challenges was designing the map for multiple zoom levels. Maps need to show different levels of detail depending on how far the user is zoomed in or out. If too many features appear at smaller scales, the map becomes cluttered and hard to read. Because of this, I had to decide which features were most important at different zoom levels. At larger scales the map shows major features like large roads or major geographic areas. As the user zooms in, more detailed elements appear such as smaller roads or specific locations.

There were also technical challenges when building the interactive map. Working with the code required understanding how map layers, styling rules, and zoom levels interact with each other. At times the code did not behave the way I expected, especially when trying to control which features appeared at certain zoom levels. I used AI tools to help troubleshoot some of these issues and to better understand how certain parts of the mapping code worked.

AI was particularly helpful when I needed examples of how to structure certain parts of the code. It helped me quickly test different solutions and understand possible fixes when something was not working correctly. However, I still had to experiment and adjust the code myself to make sure everything worked properly.

Overall this lab helped me understand both the design and technical sides of mapping. Creating a basemap requires thinking about aesthetics, readability, and user experience. At the same time it also requires solving coding challenges and understanding how digital maps work behind the scenes. The process showed me that good map design is a balance between visual design and technical implementation.
